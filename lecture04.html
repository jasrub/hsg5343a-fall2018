Cascading Style Sheet

Why CSS?

  • HTML was never intended for formatting or styling purposes but to define the content of a document
  • Once you embed presentation inside of an HTML tag, it cannot be overridden
  • The bottom line: separate the structure and content
  • Gives the designer creative control of the content. What you can change:
    • Font and font sizes
    • Color
    • Layout
    • Position
    • Borders
  • Control the layout of content on multiple devices using multiple style sheets (and thus improving accessibility)
  • Provides visual feedback to users

CSS Basics

A rule is made up of a selector with one or more declarations. Each declaration is composed by a property/value pair.

CSS Syntax

  • A CSS file contains a sequence of rules
  • Example rule: h1 { color: blue; }
    • h1 is known as the selector
    • color: blue; is known as a declaration
    • color is known as the property
    • blue is known as the value
  • The selector comes first, then the declarations are enclosed by brackets { }
  • A rule can have many declarations
  • A declaration is a property-value pair separated by a colon :
  • Declarations always ends with a semicolon ;
  • You can put one declaration on each line
  • To make a comment in a CSS, use /* COMMENT HERE */

Styling HTML with CSS

CSS can be added to HTML elements in 3 ways:

  • External - by using an external CSS file. declare in <head> section [preferred] 
    <link rel="stylesheet" href="styles.css">
  • Internal - by using a <style> element in the <head> section [meh] 
    
                                <style>
                                    body {background-color: powderblue;}
                                    h1   {color: blue;}
                                    p    {color: red;}
                                </style>
  • Inline - by using the style attribute in HTML elements [yuck] 
    <h1 style="color:blue;">This is a Blue Heading</h1>

class selector

The .class selector selects elements with a specific class attribute.
The class selector starts with a period (.) character, followed by the name of the class.

html


                        <h1 class="title" > Some Title </h1>

css


                        .title{
                            line-height: 32px;
                        }

This rule defines the line height (space between the lines) for all elements with the attribute class="title"

id selector

Styles the element with the specified id
The id selector starts with a hash (#) character, followed by the id name.

html


                        <h1 id="main-title" > Some Title </h1>

css


                        #main-title{
                            color: red;
                        }

This rule defines the text color of a tag with a specific id

id selector vs class selector

  • id selector
    • Used to specify a style for a single, unique element
    • Uses the id attribute of the HTML element. IMPORTANT: the ID cannot start with a number
    • Defined with a #
    • Each tag can only have one id
    • Each id can only appear once on an html page
  • class selector
    • Used to specify a style for a group of elements
    • Class selector is most often used on several elements
    • The class selector uses the HTML class attribute, and is defined with a .
    • In HTML, a tag can contain multiple class names, specified in the class attribute and separated by spaces
    • A class name can appear multiple times on an html page

The "Cascading" in Cascading Style Sheets

  • What if there is more than one style specified for an HTML element?
  • The styles will roll-up or "cascade" into a bigger rule
  • The last declaration wins!
  • Priorities:
    • Browser default
    • External style sheet
    • Internal style sheet
    • Inline style

Advanced Compositions

  • Descendants, separated by space
    • Example: 
      form p {...}
      - Applies to all paragraph tags inside form elements
  • Child of, separated by >
    • Example: 
      form > p {...}
      - Applies to all paragraph elements that are direct children of form elements
  • Next sibling, separated by +
    • Example: h1+strong - Applies to all <strong> elements directly following a <h1> element
  • General sibling, separated by ~
    • Example: h1~strong - Applies to all <strong> elements following a <h1> element

Children and Descendants


                        section p{
                            color: red;
                        }
                        /* Styles in red every p that is inside a section */
                        /* (a descendant. can be a "grandchild") */
                    

                        section > article > p{
                            color: red;
                        }
                        /* p that is directly inside an article that is */
                        /* directly inside a section (direct child) */

@font-face

  • font files that are located in the website folder and are imported through the CSS
  • Various browsers read various font files. In order to render a font in all the browsers, we'll need to include multiple versions of the same font (woff, otf, ttf, etc.)
  • woff is supported by the latest versions of most browsers and so is the preferred format
  • Converting fonts:
    • font squirrel
    • everything fonts

@font-face


                    @font-face {
                      font-family: 'MyWebFont';
                      src: url('webfont.eot'); /* IE9 Compat Modes */
                      src: url('webfont.eot?#iefix') format('embedded-opentype'), /* IE6-IE8 */
                           url('webfont.woff2') format('woff2'), /* Super Modern Browsers */
                           url('webfont.woff') format('woff'), /* Pretty Modern Browsers */
                           url('webfont.ttf')  format('truetype'), /* Safari, Android, iOS */
                           url('webfont.svg#svgFontName') format('svg'); /* Legacy iOS */
                    }
                    
                    body {
                      font-family: 'MyWebFont', Fallback, sans-serif;
