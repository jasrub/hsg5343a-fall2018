Variables

                    
                    let monthlyRent = 500;

                    let yearlyRent = monthlyRent * 12;
                    console.log(yearlyRent);
  • Variables are containers that you can store values in.
  • You start by declaring a variable with the const, let or var keywords, followed by any name you want to call it
  • We first declare a variable called monthlyRent
  • let is a keyword that let's javascript know we are declaring a variable
  • We use const to declare variables that can't change
  • Variable names cannot have spaces in them
  • Variable can be called almost anything. We can't use keywords
  • We use CamelCase for compound words

Variables Naming Rules

  • Generally, stick to just using Latin characters (0-9, a-z, A-Z) and the underscore character.
  • You shouldn't use other characters because they may cause errors or be hard to understand for an international audience.
  • Don't use underscores at the start of variable names.
  • Don't use numbers at the start of variables. This isn't allowed and will cause an error.
  • Make variable names intuitive, so they describe the data they contain. Don't just use single letters/numbers, or big long phrases.
  • Variables are case sensitive — so myage is a different variable to myAge.
  • Avoid using JavaScript reserved words as your variable names. So we can't use words like var, function, let, and for as variable names. Browsers will recognize them as different code items, and so you'll get errors.

Expressions and Operators

Equal (==) Returns true if the operands are equal. 3 == var1

"3" == var1

3 == '3'
Not equal (!=) Returns true if the operands are not equal. var1 != 4
var2 != "3"
Strict equal (===) Returns true if the operands are equal and of the same type. 3 === var1
Strict not equal (!==) Returns true if the operands are of the same type but not equal, or are of different type. var1 !== "3"
3 !== '3'

* To avoid unexpected behavior and hard to track bugs always prefer === and !==

The while statement creates a loop that executes a specified statement as long as the test condition evaluates to true.

Side note - shortcuts for adding 1 to variables

For Loop

                    
                        let friendsAtYourParty = 0;
                        for (let i = 0; i <= 10; i++) {
                          friendsAtYourParty++;
                        }
                        console.log(friendsAtYourParty);

The for statement creates a loop that consists of three optional expressions, enclosed in parentheses and separated by semicolons, followed by a statement (usually a block statement) to be executed in the loop. 

                        
                        for ([initialExpression]; [condition]; [incrementExpression])

The for loop repeats until the condition evaluates to false

For Loop

                        
                        for ([initialExpression]; [condition]; [incrementExpression]) {
                            statement
                        }
                        
                    
                        
                        for (let i = 0; i <= 10; i++) {
                          friendsAtYourParty++;
                          console.log("there are " + friendsAtYourParty + " friends at your party")
                        }
  1. The initializing expression initialExpression is executed. This expression usually initializes one or more loop counters.
  2. The condition expression is evaluated. If the value of condition is true, the loop statements execute. If the value of condition is false, the for loop terminates.
  3. The statement executes. To execute multiple statements, we use a block statement ({ ... }) to group those statements.
  4. If present, the update expression incrementExpression is executed.
  5. Control returns to step 2.

Infinite Loops (Oh Snap!)

Sometimes, if we mess up what's inside the control condition for the loop, we'll get a runaway loop that'll never complete. This is called an infinite loop and it'll lock up and crash our code and browser. Something like this:

                        
                        //Warning!! Will crash the browser if you try to run this!
                        let friendsAtYourParty = 1;
                        while (friendsAtYourParty > 0) {
                          friendsAtYourParty = friendsAtYourParty + 1;
                        }
                        console.log(friendsAtYourParty);

Since we're adding one to friendsAtYourParty each time, and the loop will continue each time until it's less than zero, that condition will never happen. Thus it'll continue going until it crashes our code. Be careful of these.

Functions

Functions let us define the code once, and use it many times.

                    
                    function addTwo(number) {
                      return number + 2;
                    }

                    const finalAnswer = addTwo(5);
                    console.log(finalAnswer);

A JavaScript function is defined with the function keyword, followed by a name, followed by parentheses ().
Function names can contain letters, digits, underscores, and dollar signs (same rules as variables).
The parentheses may include parameter names separated by commas:(parameter1, parameter2, ...)
The code to be executed, by the function, is placed inside curly brackets: {}

Functions

When JavaScript reaches a return statement, the function will stop executing.
Functions often compute a return value. The return value is "returned" back to the "caller"

Functions

The way call a function is you add parenthesis () to the end of it, like this: someFunctionName(). If you see parenthesis after a variable name, you instantly know that that's a function. Inside of the parenthesis go the parameters. These variables will be passed to the function that is being called in the order that you put them there. Example: 

                    
                    const myHomeCity = "Brooklyn";
                    const myHomeState = "New York";
                    const myHomeCountry = "USA";

                    function logOutYourHome(city, state, country) {
                      console.log(`You are from ${city}, ${state} ${country}.`);

                    logOutYourHome(myHomeCity, myHomeState, myHomeCountry);
                    logOutYourHome("Torino", "Pimonte", "Italy"); // also works

Scope

Scope

Scope determines the accessibility (visibility) of variables.

                    
                    function addFive(number) {
                        const someVariable = "you can't see me outside this function";
                        return number + 5;
                        console.log(someVariable)
                    }

                    addFive(10);
                    console.log(someVariable); // Causes error

This is not going to work. someVariable is inside of the addFive scope and once addFive completes, it throws someVariable away since it's now out-of-scope.

Scope

The lifetime of a JavaScript variable

  • The lifetime of a JavaScript variable starts when it is declared.
  • Local variables are deleted when the function is completed.
  • In a web browser, global variables are deleted when you close the browser window (or tab), but remain available to new pages loaded into the same window.
