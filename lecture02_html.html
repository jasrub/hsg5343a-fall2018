HTML

Hyper Text Markup Language

  • Creates the structure of a web page
  • Gives Semantic meaning to the content

Brief History

  • Tim Berners-Lee, then a contractor at CERN, specified HTML and wrote the browser and server software in late 1990.
  • IETF began formally specifying HTML in 1993, and after several drafts released version 2.0 in 1995
  • In 1996, the W3C (founded by Berners-Lee) took over the HTML work and published the HTML 3.2 recommendation a year later
  • HTML 4.0 was released in 1999 and became an ISO standard in 2000
  • In 2004, WHATWG unhappy with slow progress by W3C started formulating a new HTML spec
  • The two organizations released the first draft for HTML 5 spec in 2008 and the final standard in 2014
  • HTML 5.0 is the current and latest standard

HTML

Hyper Text Markup Language

  • A set of "tags" are added to a page define its structure
  • Documents are "linked" to other documents
  • Tree-like representation of a page structure
  • Most whitespace is insignificant in HTML. That is, if you have a long run of whitespace, it is all collapsed into a single space

Anatomy of an html element

Structure of an HTML Page

Base Tags

  • There is a specific structure of tags that needs to exist in each HTML document
  • The file always starts with an opening <html> tag and ends with a closing </html> tag.
  • The !DOCTYPE tag tells the browser this is an HTML file and must be the first thing at the top of every html file
  • The <html> element will include 2 direct children tags: <head> and <body>

                        <!DOCTYPE  html>
                        <html>
                            <head>
                            </head>
                            <body>
                            </body>
                        </html>

Base Tags

The <head> tag includes information about the page (e.g., title, abstract, stylesheet declarations, keywords).Its content does not appear in the rendered document. The title is displayed on title bar of the web browser and it is used as the name of your bookmark. 

                        <!DOCTYPE html>
                        <html>
                            <head>
                                <title> Moby Dick </title>
                                <meta charset="UTF-8">
                            </head>
                            <body>
                            </body>
                        </html>

Base Tags

The <body> tag includes the content of the page. There are many ways to create content. Tags that will be rendered on the screen go inside the <body> tag. 

                        <!DOCTYPE html>
                        <html>
                            <head>
                            </head>
                            <body>
                                <h1> Moby Dick - Chapter 1 </h1>
                                <p> Ahab stood up to the blast...</p>
                            </body>
                        </html>

Block-level elements

contain an entire large region of text

  • <p></p> - Paragraph
  • <h#></h#> - Header
  • <hr/> - Horizontal rule; a long horizontal line
  • <ul></ul> and <li></li> - Unordered list; <ul></ul> is a single block of bulleted-list items; each <li></li> inside the <ul> is an item of the list
  • <ol></ol> and <li></li> - Ordered list; <ol></ol> is a single block of ordered-list items; each <li></li> inside the <ol> is an item of the list
  • <blockquote></blockquote> - A block of quotation
  • <div></div> - Division; generic block element (more on this in CSS)
  • <table></table> - Table
  • <form></form> - Web form
Inline-level elements

affect a small region of text; nested inside a block-level element

  • <br/> - A forced line break
  • <code></code> - Display text in a fixed-width font (Courier). Makes text look like source code.
  • <a></a> - Anchor; a link to another page. Use href attribute to specify the destination URL
  • <img /> - Image
  • <b></b> - Presentation: bold (old)
  • <strong></strong> - Presentation: strongly emphasized text (bold)
  • <i></i> - Presentation: italics (old)
  • <em></em> - Presentation: emphasized text (italics)
  • <u></u> - Presentation: underline
  • <sub></sub> - Presentation: subscript
  • <sup></sup> - Presentation: superscript
  • <span></span> - Generic inline element (more on this in CSS)

More on HTML Tags and Attributes

  • Most inline-level elements stay on the same line,
    and most block-level automatically create new lines.
  • Some tags can contain (multiple) attributes.
    You can place the attributes in any order. Example:
    <a href="http://www.google.com" target="_blank">Click here</a>
  • Some tags can be open and closed in one tag
    (i.e., no matching end tag). Examples:
    • <br/>
    • <img src="krusty.jpg" width="200" height="200" alt="Krusty the Clown" />
  • Nesting tags: very important to nest tags correctly
    • Closing tag must match the most recently opened tag
    • Example:
    <p>Remember folks, there is <em>no class</em> on <strong>Thursday, February 21st</strong>!</p>

Common Errors in HTML

  • Not closing a tag
  • Not closing a list
  • Improper nesting (document will likely still be rendered, but ugly)
  • Forgetting to put double-quotes around values of attributes
  • Forgetting important attributes for tags (e.g., alt for an image)
  • Using upper-case / lower-case tag names interchangeably
  • Hardly anything is enforced in HTML. Even if there is an error, the page will most likely still render!
